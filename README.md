# Webセキュリティ 実装課題2 - セッションベース認証・認可ウェブアプリケーション

本アプリケーションは、講義教材 `web-sec-playground-2` をベースに構築した、Next.js (App Router) によるセキュアなウェブアプリケーションです。
情報セキュリティの根本原則に基づき、徹底的なクリーンアップ、堅牢な防御属性の設定、および実用的な独自UI機能を実装しています。

---

## 1. 選択した認証方式とクリーンアップ

本プロジェクトでは、標準要件である **「セッションベース認証」** を選択・実装しています。

### 減点リスクの完全排除（不要コードの網羅的削除）
課題の指示に厳格に従い、実装の透明性と保守性を高めるため、以下の不要コードを型定義・コメントに至るまでプロジェクトから**完全削除**しました。
* **使用しない認証方式**: トークンベース認証（JWT）に関連するヘルパー・ロジック・型定義をすべて削除（`createJwt.ts`, `verifyJwt.ts`, `jwtFetcher.ts` 等）。
* **不要なドメインロジック**: 「ニュース（news）」および「ショップ（shop）」に関連するコンポーネント、APIルート、型定義（`CartItem.ts`, `Product.ts`, `Region.ts` 等）の完全なクリーンアップ。

## 2. ガチガチにセキュアな設計（情報セキュリティ要件の達成）

審査の観点である「ガチガチにセキュアな設計」を満たすため、バックエンドおよびクッキー制御に以下の防御策を講じています。

### ① パスワードの安全な一方向ハッシュ化保存 (`bcryptjs`)
* **実装詳細**: ユーザー登録（サインアップ）の際、ユーザーが入力した平文のパスワードをそのまま保存せず、`bcryptjs` ライブラリを用いて強力にハッシュ化しています。
* **攻撃耐性**: 強固なストレッチング（ソルトラウンド: `10`）を適用することで、万が一データベース（SQLite）が窃盗・漏洩した場合でも、レインボーテーブル攻撃やブルートフォース攻撃（総当たり攻撃）によって平文パスワードがリバースエンジニアリングされるリスクを構造的に排除し、データの**機密性 (Confidentiality)** と **完全性 (Integrity)** を担保しています。

### ② 厳格なCookie（セッションID）属性の設定
セッション管理用クッキー `session_id` には、最新のWebセキュリティのベストプラクティスに基づいた属性を厳密に設定しています。
* `httpOnly: true`: JavaScriptの `document.cookie` からのアクセスをブラウザレベルで遮断します。万が一アプリケーション内にクロスサイトスクリプティング（XSS）の脆弱性が存在した場合でも、攻撃者によってセッションIDクッキーが窃盗（セッションハイジャック）されるリスクを100%防御します。
* `sameSite: "strict"` / `"lax"`: クロスサイト・リクエスト・フォージェリ（CSRF）攻撃を完全に遮断します。第三者の悪意あるサイトを経由して、ユーザーのブラウザから勝手に認証クッキー付きのリクエストが送信されるのを防ぎ、意図しない認可操作を防止します。
* `path: "/"`: セッション範囲をアプリケーション全体に限定し、安全なスコープ管理を行います。

## 3. 教材未実装の独自機能（2機能実装）

セキュリティ設計および利便性の向上を目的とし、拡大解釈を含む独自の認証・認可関連機能を2つ以上実装しました。

### 機能①：サインアップ時における確認用パスワード要求UI機能
* **概要と意図**: アカウント作成時に、パスワードのタイポ（入力ミス）による予期せぬアカウントロックやログイン不能状態を防ぐため、確認用入力欄を必須としました。
* **設計と堅牢性**: フロントエンド専用の Zod バリデーションスキーマ（`frontendSignupSchema`）を独自定義。`.refine()` メソッドを駆使し、クライアントサイドで1文字入力されるごとに「パスワード」と「確認用パスワード」の完全一致をリアルタイムで厳格に検証します。
* **防御的UI**: 条件を満たさない（不一致、または最小文字数未満）状態では、登録ボタンを物理的に無効化（`disabled`）すると同時に、適切なエラーメッセージを表示して不正なリクエストがサーバーへ送られるのを防ぎます。

### 機能②：パスワードの表示・非表示（目玉マーク）トグル切り替え機能
* **概要と意図**: ログイン画面およびサインアップ画面の全パスワードフォームにおいて、入力内容の可視性をユーザーが動的に制御できるUIを実装しました。
* **設計**: Reactの `useState` を用いて、各入力欄の表示状態フラグをコンポーネント内に閉じ込めて管理。`FontAwesomeIcon` を用いたスタイリッシュな目玉アイコン（`faEye` / `faEyeSlash`）を配置しました。
* **防御的UI**: アイコンをクリックすることで、HTMLの入力属性を `type="password"`（背後からのショルダーハック/盗み見防止）と `type="text"`（正確な入力の確認）の間でトグル切り替えさせ、安全かつ快適なパスワード入力を強力にサポートします。
